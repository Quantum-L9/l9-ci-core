# First-Order Thinking Kernel

## Cardinal Question

What is the simplest change that delivers the most functional value?

## Use When

Consult before:

- choosing one Skill versus multiple Skills
- adding references, scripts, or assets
- accepting a broad user plan
- performing large rebuilds
- prioritizing build work
- expanding scope

## Five Gates

### 1. Impact Gate

Determine whether the work adds functional capability or only reorganizes material.

Functional work outranks cosmetic work unless the user explicitly chooses otherwise.

### 2. Effort/Coverage Gate

Compare work required to value covered.

Prefer designs that affect more future use cases with fewer files, fewer rules, and less maintenance.

### 3. Dependency Gate

Do unlocking work before dependent or cosmetic work.

If one decision enables many later decisions, surface it.

### 4. Rabbit-Hole Gate

Detect expansion beyond the original ask.

If the workstream grows into unrelated domains, checkpoint and re-center on the shipping objective.

### 5. Prompt-Quality Gate

If the user message mixes topics, decompose and prioritize by impact before building.

## Hard Rules

- Surface a better simpler path before generating artifacts.
- Do not confuse thoroughness with leverage.
- Do not let structure work outrank functional value.
- Do not add files because the folder shape looks impressive.
- Quantify effort and coverage when useful.

## Validation Gates

- Highest-value path identified.
- Complexity justified.
- Sequencing checked.
- Scope creep checked.
- User intent preserved.
